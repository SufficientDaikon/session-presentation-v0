# v0.1 Plan — Responsive-First Presentation Overhaul

> **Date**: 2026-04-16
> **Status**: Ready for implementation
> **Reviewed**: 2 passes (gaps found → addressed, architecture validated)

---

## Problem

The presentation has a single `@media (max-width: 768px)` breakpoint — basically useless. reveal.js scales everything uniformly via CSS transform, which means on small screens text shrinks proportionally and on ultra-wide screens there's wasted space. We want **responsive-first** — not mobile-first, not desktop-first. Content should adapt intrinsically to ANY screen size.

---

## Approach: BYOL + Fluid CSS + Fitty + Pretext

### The Stack (3 libraries + modern CSS)

| Layer                 | Tool                                                | Stars  | Role                                                                                        |
| --------------------- | --------------------------------------------------- | ------ | ------------------------------------------------------------------------------------------- |
| Layout engine         | reveal.js BYOL (`disableLayout: true`)              | 68k    | Turns off uniform scaling, gives us full viewport control                                   |
| Display text scaling  | **Fitty.js** 2.4.2                                  | 3.8k   | Auto-scales hero titles, section numbers to fill their container                            |
| Text measurement      | **Pretext** 0.0.5 (@chenglou)                       | 44.2k  | DOM-reflow-free text measurement for balanced layout, overflow detection, smart card sizing |
| Responsive foundation | CSS `clamp()` + Container Queries + Intrinsic Grids | native | Fluid typography, fluid spacing, component-level adaptation                                 |

### Why this combination is SOTA

- **Fitty** = "make this text as big as possible" (hero titles, stats, display text)
- **Pretext** = "measure text without DOM reflow and make smart layout decisions" (balance text, detect overflow, shrink-wrap cards, prevent orphans)
- **CSS clamp()** = continuous scaling without breakpoint jumps (typography + spacing)
- **Container Queries** = component-level responsive (cards adapt to their own container, not the viewport)
- **Intrinsic Grids** = `auto-fit` + `minmax()` — columns self-determine based on available space
- **BYOL mode** = slides fill viewport naturally instead of being a scaled-down 1280×720 box

### Why this beats the default scaling model

- **Default**: 1280×720 canvas scaled down → on phone, everything shrinks uniformly → tiny text, tiny buttons
- **BYOL**: slides fill viewport → text/grids/spacing adapt fluidly → readable at every size, tappable buttons, natural reflow

### CDN Sources (verified working 2026-04-16)

- Fitty: `https://cdn.jsdelivr.net/npm/fitty@2.4.2/dist/fitty.min.js` (3.8KB)
- Pretext: `https://esm.sh/@chenglou/pretext@0.0.5` (ESM module)

## Key CSS Architecture

### Fluid Typography Scale

```css
--fs-hero: clamp(2rem, 5vw + 1rem, 4.5rem);
--fs-h1: clamp(1.75rem, 3.5vw + 0.5rem, 3rem);
--fs-h2: clamp(1.25rem, 2.5vw + 0.5rem, 2.2rem);
--fs-h3: clamp(1rem, 1.5vw + 0.5rem, 1.5rem);
--fs-body: clamp(0.875rem, 1vw + 0.25rem, 1.1rem);
--fs-small: clamp(0.75rem, 0.8vw + 0.25rem, 0.9rem);
--fs-xs: clamp(0.625rem, 0.6vw + 0.2rem, 0.8rem);
--fs-stat: clamp(1.5rem, 4vw + 0.5rem, 2.8rem);
```

### Fluid Spacing Scale

```css
--space-xs: clamp(0.25rem, 0.5vw, 0.5rem);
--space-sm: clamp(0.5rem, 1vw, 1rem);
--space-md: clamp(1rem, 2vw, 2rem);
--space-lg: clamp(1.5rem, 3vw, 3rem);
--space-xl: clamp(2rem, 4vw, 4rem);
--slide-pad: clamp(1rem, 3vw, 3.75rem);
```

### Intrinsic Grid

```css
.card-grid {
  grid-template-columns: repeat(auto-fit, minmax(min(280px, 100%), 1fr));
}
```

### Container Queries

```css
.reveal .slides section {
  container-type: inline-size;
}
@container (max-width: 600px) {
  /* stack layouts */
}
@container (min-width: 900px) {
  /* wide layouts */
}
```

### Key reveal.js Feature Used

`disableLayout: true` — documented at [revealjs.com/presentation-size](https://revealjs.com/presentation-size/)

> "If you want to disable the built-in scaling and centering and Bring Your Own Layout, set `disableLayout: true`. That will make your slides cover 100% of the available page width and height and leave the responsive styling up to you."

## Notes

- Pretext is by chenglou (creator of React Motion) — 44.2k stars, MIT license, TypeScript
- Pretext demos show: tight bubbles, masonry, editorial engine, balanced text — all directly applicable to our slides
- Container queries have 93%+ browser support (2025+) — safe to use
- Fitty handles its own resize listener internally via `fitty.observeWindow`
- Pretext `prepare()` needs canvas context — works in all modern browsers
- ESM via esm.sh is production-grade (used by Deno, etc.)
- `dvh` (dynamic viewport height) handles mobile browser chrome appearing/disappearing
- No build step needed — everything loaded via CDN, static site deployment unchanged
